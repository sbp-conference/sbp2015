Big Data Analytics for Behavioral Modeling

Abstract:
Due to the pervasive adoption of the ubiquitous cloud, distributive grid and mobile computing, data collection has grown exponentially in every industry, research area, and government sector. Making sense and managing the life cycle of big data becomes a very complex topic. For instance, the use of Web 2.0 technologies and applications, in particular social media, has efficiently gathered a large volume of timely feedback and opinions from a diverse, global population (Fan and Gordon, 2014). In fact, data about people, their behaviors and communications, and the events they experience can be processed and used to detect population trends, predict behavior and inform decisions (Ribarsky et al., 2013; Chen et al., 2012). This tutorial will introduce participants to Splunk, a commercial software tool used to search, monitor and analyze big data, to give participants an opportunity to engage with data analytical tools and complete hands-on exercises. The hands-on exercises will start with an introduction to an experimental big data set, followed by a discussion about the nature of data and some analysis perspectives and techniques. These exercises will allow participants to identify and quantify the domains of specific research areas (e.g., anonymized healthcare, social, or business data) using a “big data” set that addresses their research questions. Participants will be placed in groups where they will perform an analysis on a prepared data set. Each group will have a different analytical focus. This analysis will require participants to parse out probable variables and make comparison and inference about users’ behavioral patterns and/or motivation models as displayed by actors collectively. Participants will be invited to demonstrate their findings and share their views after the completion of these hands-on exercises.

Technology Requirements:
Participants are required to bring their own laptops. Please download and install Splunk prior to the tutorial: http://dev.splunk.com/page/developer_license_sign_up

A Tutorial on Text and Network Analysis Methods

Abstract:
This tutorial will introduce audience members to the basics of social network analysis and unsupervised text analysis methods. Topics covered include sources of network data, social network metrics, and an overview of text analysis techniques that can be used to perform computational social science. Students will gain experience using a range of social network analysis techniques, and exposure to machine learning and statistical algorithms that can be used to perform in-depth analysis.

Specifically, we will provide an introduction to basic natural language processing algorithms such as Latent Semantic Analysis and Latent Dirichlet Allocation. Detailed examples from the instructor’s research in the Food and Drug Administration will be provided.

Agenda:

  • Section I: Social Network Analysis (50 minutes)
    • What is a Network?
    • What are network data?
    • Basic Network Metrics
    • Exploring Network Metrics using Gephi
  • 10 minute break
  • Section II: Text Analysis (50 minutes)
    • Why text analysis for computational social science?
    • Where does the text come from?
    • Two types of algorithms: supervised and unsupervised
    • Example of supervised data analysis – Amazon Mechanical Turk and Influenza Tracking on Twitter
    • Example of unsupervised data analysis – Latent Semantic Analysis 
    • The “bag of words” approach to meaning
  • 10 minute break
  • Section III: Network and Text Analysis Combined (60 minutes)
    • Latent Dirichlet Allocation – LSA 2.0
    • Using LDA to find topics organically in text
    • Using topics from LDA to build a network
    • Case Study in the US FDA Advisory Panels

Laptops are not required but may be helpful for those wishing to follow along in the network analysis section. Software link: https://gephi.github.io/

Behavioral Mining, Cultural Differences and Social Network Analysis in Massive Online Games

Abstract:
Massive Online Games (MOGs) refer to massive online persistent environments (World of Warcraft, EVE Online, EverQuest etc) shared by millions of people . In general these environments are characterized by a rich array of activities and social interactions with a wide array of behaviors e.g., cooperation, trade, quest, deceit, mentoring etc. Such environments allow one to study human behavior at a level of granularity where it was not possible to do so previously. Given the challenges associated with analyzing this type of data traditional techniques in data mining and social network analysis have to be extended with insights from the social sciences. The tutorial will cover predictive and generative models in the study of MOGs. Additionally we will cover some SNA techniques which are more appropriate for MOGs given the multi-dimensionality of the data (P*/ERGM Models, IR Based Network Analysis, Hypergrah based Techniques, Coextensive Social Networks etc). Based on our published work in this area we also describe the various ways in which MOGs exhibit similarities to the real world e.g., economic behaviors, clandestine behaviors, mentoring etc). Lastly we describe the scope and limitations of analysis of MOGs based on limitations in data collection, availability and ethical concerns. An overview of differences in human behavior between Chinese and American MMOs is also given.

Introduction to social network and spatial data in R and Python to social scientists

Abstract:
I introduce the effective collection, organization and exploration of spatial data (maps) and social network data to social scientists. Detailed templates in Python and R will be shared that automate the processes mentioned above. A particular emphasis will be given to data collected from the web, the visualization of data in networks and/or maps. I finally present steps for publication of reports via the tools offered in Python (IPython and Github) and R (RStudio, Github and RPubs), including brief explanations on how interact with UCINET, Gephi, Netlogo and QGIS.

Length of Tutorial: 3 hours.

A list of references/resources to be covered in the tutorial
- Pre-Session (via email): notes on preparing the laptop to make python and R work.
- Introduction: Using IPython+Github and R+Rstudio+RPubs
- Collecting Data: scrapping data from websites using R and Python
- Preparing Data: cleaning and formatting data using R and Python
- Basic Data Exploration using R and Python
- Social networks: exploring and visualizing social network data in Python and R
- Spatial data: Preparing informative maps using Python and R
- Publishing results
- Comments to combine Python, R, QGIS and Netlogo

Requirements
Class is prepared for a target audience of social scientists with no experience in data science (It is assumed they have only worked with Excel spreadsheets, SPSS or STATA). The must bring their laptop for the tutorial, and should have installed:
Python Canopy – only the academic version for 2.7
Anaconda for Python 2.7
R and RStudio.

Introduction to Dynamic Network Analysis

Abstract:
We discuss dynamic network analysis and how it can be useful both for instantiating simulations and for novel simulation outputs. The tutorial focuses on providing attendees grounding in network analysis, allowing attendees to incorporate network data as inputs or as outcomes of their simulation tools. We first introduce key network analysis concepts and statistics including centrality measures and brokerage. To ground these concepts and provide some intuition for thinking about different types of centrality measures, we run a short hands-on analysis of network data in ORA based on intelligence from the Revolutionary War. We then introduce the data structures commonly used in network analysis and help attendees import their own data into ORA. We then introduce attendees to grouping algorithms including clique identification, Concor, and Newman-Girvan. We conclude with additional methodologies to compare networks such as QAP. These methods are particularly useful for comparing simulation inputs and outputs, and inform analysis of simulation outputs. Attendees are encouraged to bring their own data, but sample data will be provided.
